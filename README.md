# CALculeted-
calc
Проект: «Калькулятор с обратной польской записью (RPN)»
Суть:
Go-программа – это интерфейс (ввод пользователя, вывод результата).
C-программа – это вычислительный движок (парсит выражение и считает).

Они общаются через стандартный ввод/вывод (Go запускает C-программу как подпроцесс и передаёт ей строки через stdin / stdout). Это самый простой способ межпроцессного взаимодействия без возни с сокетами.

🟢 Логика на Go (интерфейс)
Запускает C-программу как дочерний процесс с каналами stdin и stdout.

В бесконечном цикле:

Выводит приглашение >.

Читает строку от пользователя (например, 3 4 + или 10 2 /).

Если введено exit – завершает работу (и убивает дочерний процесс).

Отправляет строку в stdin C-программы.

Ждёт ответа из stdout C-программы и выводит результат.

Обрабатывает ошибки (если C-программа вернула ошибку, показать сообщение).

🔵 Логика на C (вычислитель)
Читает из stdin построчно (используя fgets).

Для каждой строки:

Разбивает на лексемы (числа и операторы + - * /) по пробелам.

Использует стек (массив double) для вычисления RPN:

Если лексема – число → кладёт в стек.

Если оператор – извлекает два верхних числа, выполняет операцию, кладёт результат обратно.

После обработки всей строки выводит результат в stdout (с новой строкой) и завершает работу (либо продолжает читать следующие строки – проще завершаться после каждого выражения, но тогда Go каждый раз перезапускает процесс; лучше, чтобы C-программа работала в цикле, пока не получит специальную команду quit).

Для простоты сделаем так: C-программа читает строки в цикле, пока не встретит quit (или EOF). После каждого выражения выводит результат.

День 1:

День 2:

Написать на Go код, который запускает скомпилированную C-программу через exec.Command.

День 3 (запасной):

Сделать небольшую документацию (README).

💡 Почему этот проект крут для подростков?
Понятный алгоритм – RPN легко реализовать на стеке, не нужно писать сложный парсер инфиксных выражений.

Реальное взаимодействие – они узнают, как запускать внешние программы и общаться через pipe.

Разделение ответственности – каждый язык делает то, что умеет лучше: C – быстрые вычисления, Go – удобный I/O и управление процессами.

Можно расширять – добавить новые операторы, поддержку скобок, или даже превратить в сетевой сервис (заменить pipe на TCP) – это уже следующий уровень.

🛠 Что нужно знать для реализации
C: работа с stdio.h, строками (string.h), динамический массив или простой статический стек.

Go: пакеты os/exec, bufio, io, работа с каналами StdinPipe / StdoutPipe.

Общее: компиляция C-программы (gcc calc.c -o calc), а Go-программа будет запускать этот бинарник.
